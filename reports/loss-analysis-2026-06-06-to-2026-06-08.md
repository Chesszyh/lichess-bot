## Takeaway

The current fast-game weakness is not just time management. A repeated chess-strength pattern is that the bot, as Black, follows sharp bot-vs-bot opening-book lines deep enough to enter high-risk Najdorf structures against strong engines.

The speed-specific polyglot depth work is therefore aligned with the observed losses. This turn tightens the implementation around that feature by:

- Validating `max_depth_by_speed` speed keys and non-negative integer depths.
- Preserving a configured depth of `0`, so a profile can fully disable the local polyglot book for a specific speed instead of silently falling back to the base book depth.

## Follow-Up on 2026-06-08

After the blitz book-depth change was loaded, a fresh scan of the latest 80 Lichess games and the local 7-day PGN set
still showed the same black-side concentration:

- Latest 80 Lichess games: `10` losses, `9` as Black, with `6` Sicilian/Najdorf-family losses and `2` Semi-Slav losses.
- Local 7-day PGNs: `583` games, `102` losses, `90` as Black.
- Local 7-day PGNs: top loss opening remained Najdorf English Attack with `13` losses.
- Local 7-day PGNs: lower-rated bot draws existed (`43`), but were opening-dispersed; the loss cluster was more specific.

Book inspection explains why the cluster persisted. The bot-vs-bot profile used only `komodo.bin` with `best_move`.
For `1. e4`, that book offers `...c5` as the top move, then continues through `...d6`, `...cxd4`, and `...Nf6`.
The fast depth cap stops before the full Najdorf, but it still seeds the same Sicilian structure every time.

Private config was therefore changed to use `rodent.bin` with `uniform_random`, `min_weight: 90`, and the existing
`bullet: 4` / `blitz: 4` depth cap for bot opponents. In the local book, this keeps only high-weight replies while
making `1. e4` choose among `...c5`, `...e5`, and `...c6` instead of forcing `...c5`. This is a low-resource change:
no local engine match was run, no outgoing challenge cadence was increased, and no Lichess online book requests were
enabled.
